# Webhook Verification Guide

NoxPay signs every webhook payload with an HMAC-SHA256 signature to ensure authenticity and integrity. You must verify this signature before processing the event.

## 🔑 Required Components
- **Webhook Secret**: Your unique secret (starts with `whsec_`), obtainable from the Merchant Dashboard.
- **X-NoxPay-Signature**: The HMAC signature provided in the request headers.
- **X-NoxPay-Timestamp**: The unix timestamp provided in the request headers.

## 🛡️ Verification Logic
The signature is generated by hashing the string `{timestamp}.{payload}` using your webhook secret.

### Python Example (FastAPI/Flask)
```python
import hmac
import hashlib
import json

def verify_noxpay_webhook(payload_body, signature, timestamp, secret):
    # 1. Reconstruct signing input
    signing_input = f"{timestamp}.{payload_body}".encode('utf-8')
    
    # 2. Compute expected HMAC
    expected_signature = hmac.new(
        secret.encode('utf-8'),
        signing_input,
        hashlib.sha256
    ).hexdigest()
    
    # 3. Securely compare (prevents timing attacks)
    return hmac.compare_digest(expected_signature, signature)
```

### Node.js Example (Express)
```javascript
const crypto = require('crypto');

function verifyNoxPayWebhook(payloadBody, signature, timestamp, secret) {
    const signingInput = `${timestamp}.${payloadBody}`;
    const expectedSignature = crypto
        .createHmac('sha256', secret)
        .update(signingInput)
        .digest('hex');
    
    return crypto.timingSafeEqual(
        Buffer.from(signature),
        Buffer.from(expectedSignature)
    );
}
```

## ⚠️ Best Practices
1. **Replay Protection**: Check if the `X-NoxPay-Timestamp` is within a reasonable window (e.g., last 5 minutes) to prevent replay attacks.
2. **Raw Body**: Always use the **raw** request body (string/bytes) for verification, as JSON parsing can alter spacing and break the hash.
3. **Secret Security**: Treat your webhook secret as a sensitive credential. Never expose it in client-side code.
